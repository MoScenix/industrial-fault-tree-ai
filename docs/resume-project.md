# industrial-fault-tree-ai 项目简介

## 项目介绍
基于 Go + Hertz + Kitex + eino 构建的工业设备故障树智能生成与辅助分析平台。系统围绕工业文档解析、故障知识检索、故障树版本管理、AI 对话式辅助编辑等场景展开，支持工程师基于项目资料快速构建、修改和确认故障树，并通过微服务架构完成 AI、文档、图管理、用户体系等核心能力解耦。

## 核心技术
Go、Hertz、Kitex、eino、GORM / sqlx、Redis、Docker、Chromedp、Prometheus、Jaeger、Milvus

## 技术亮点
- 基于 `Kitex + Hertz` 拆分 AI、文档、图管理、用户等微服务，并通过统一 `IDL`、`rpc_gen` 和公共组件沉淀服务间调用规范，提升模块边界清晰度与后续扩展效率。
- 基于 `eino` 设计 AI Agent 能力边界，抽象项目上下文获取、RAG 检索、临时图读取与写回等工具接口，支持对工业场景下故障树生成、校验和对话式辅助编辑的统一接入。
- 设计面向工程编辑场景的图管理体系：数据库仅维护项目基础信息，版本与临时编辑态全部落在文件系统，通过 `tmp + versions + current` 组织方式支撑 AI 与工程师协同编辑、版本保存与回退。
- 构建文档管理服务，支持个人/项目 PDF 解析、文档分片与检索接口预留，并规划使用 `Milvus + Eino` 承接语义检索链路，为工业知识问答和故障树证据引用提供底层支持。
- 引入 `Redis + 本地缓存` 的多级缓存策略，对项目信息、聊天记录等高频查询路径做缓存优化，降低数据库压力并提升复杂协作场景下的接口响应性能。
- 统一服务配置、启动、监控与容器化管理方式，结合 `Docker`、`Prometheus`、`Jaeger`、公共中间件与服务脚手架，降低多微服务项目的运维与排障成本。
